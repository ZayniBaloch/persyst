Add to your Agent Config

Install and configure Cursor automatically:

npx persyst-mcp init --mcp cursor

This single command writes your global Cursor MCP settings, sets up file watch directories, and creates local workspace memory rules in 10 seconds.

Note on Active Retrieval: Persyst has two modes: Passive Recording (the watcher automatically extracts and saves memories from your conversations in the background) and Active Retrieval (your agent calls search_memories or get_optimized_context to fetch context). The IDE does not automatically inject memories into prompts unless configured to do so via workspace rules (e.g. .cursorrules, .windsurfrules, .agents/AGENTS.md) or custom system prompt configurations.
Configure Other Coding Environments (Claude Code, VS Code, Aider, etc.)

Install automatic pre-prompt memory injection hooks with one command:

npx persyst-mcp setup

This configures Claude Code's ~/.claude/settings.json to load your memory on startup and retrieve context automatically on every prompt.

Alternatively, configure it manually as a standard MCP server in your ~/.claude.json.

1. Add the standard MCP server in Cursor under Settings > Features > MCP:

  1. Click + Add New MCP Server
  2. Set Name to persyst
  3. Set Type to stdio
  4. Set Command to npx -y persyst-mcp

2. Initialize automatic workspace memory rules:

npx persyst-mcp init

This creates a .cursorrules file that instructs the agent to automatically query memory and record milestones.

Cline / Roo Code MCP Settings

Add the server configuration to your cline_mcp_settings.json settings file:

{
  "mcpServers": {
    "persyst": {
      "command": "npx",
      "args": ["-y", "persyst-mcp"]
    }
  }
}

Configure Workspace Rules & Swarm Settings:

npx persyst-mcp init

This creates a .persystrules.md file that teaches your agent how to query and store memories. To enable agentic swarm isolation, instruct your agents to pass their unique agent name as agent_id and set shared: false for private memories.

Run Aider with the automatic memory-injection wrapper:

npx persyst-mcp aider --model anthropic/claude-3-5-sonnet

This spawns Aider, automatically connects to Persyst, ingests git history, and injects relevant memories into your prompt inputs.

Alternatively, run standard Aider with standard MCP: aider --mcp-server persyst:npx -y persyst-mcp

Developer SDK & REST API

Build programmatic integrations using our first-party Node.js SDK and zero-dependency HTTP Gateway.

Install the package locally in your project:

npm install persyst-mcp

Initialize the SDK in Library Mode (runs local SQLite-vec engine offline) or Gateway Mode (connects to running HTTP gateway):

import { Persyst } from 'persyst-mcp/sdk';

// 1. Library Mode (Offline Direct SQLite Access)
const sdk = new Persyst({
  mode: 'library'
});

// 2. Gateway Mode (Connect to HTTP Server)
const sdk = new Persyst({
  mode: 'gateway',
  host: '127.0.0.1',
  port: 4321
});

// Track a developer event or milestone
const result = await sdk.track({
  content: "User prefers dark mode for all dashboards.",
  workflow: "agent-alpha",
  sessionId: "session-100",
  shared: true
});

// Retrieve compiled, optimized context tailored by intent/workflow
const context = await sdk.context({
  query: "Theme configuration",
  workflow: "agent-alpha",
  maxTokens: 2000
});

console.log(context);

Integrate with any programming language using zero-dependency HTTP endpoints:

POST /remember — Bypasses cognitive noise filter for explicit user commands

curl -X POST http://127.0.0.1:4321/remember \
     -H "Content-Type: text/plain" \
     -d "Remember to backup sqlite database every night at 2 AM"

POST /context — Fetch optimized context prompt for LLM injection

curl -X POST http://127.0.0.1:4321/context \
     -H "Content-Type: application/json" \
     -d '{
       "query": "deployment timezone",
       "max_tokens": 1500,
       "agent_id": "agent-alpha"
     }'

GET /compliance/export — Export cryptographically signed audit trail logs

curl http://127.0.0.1:4321/compliance/export?format=json

GET /health — Check server status and active memory count

curl http://127.0.0.1:4321/health
